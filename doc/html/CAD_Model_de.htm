Codes für Modelle

Modelle benutzen den Kennbuchstaben M








M internal Model



Benutzen von internen Submodels im CAD-Modus:
  Ein internes Submodel positionieren.


Submodel
  - Auswahl eines Submodels aus Liste (oder durch mittlere Maustaste)
  - oder Selektion eines bereits vorhandenen Submodels ()
  
Position,Orientation
  Nullpunkt und Ausrichtung des Submodells

[Scale]: Vergrösserungsmaßstab; Default ist 1

Beispiel:
# Internes Submodell "SM1", 45 Grad um Z-Achse gedreht, Scale=2
M20="SM1" R(P(100 0) 45) VAL(2)



- Die Modelbezeichnung darf keinen Pfad (und kein Pfadtrennzeichen "/" oder "\") enthalten.
- Alle internen Submodels werden im Modelfile abgespeichert.
- Während der Bearbeitung werden Submodelle in {base}/tmp/Model_{submodelname} zwischengespeichert.
- Alle exportierten Objekte des Submodel werden automatisch importiert.
   Der Text einer importierten Note wird neu abgfragt.


Benutzen von internen Submodels im MAN-Modus:
  durch Key-In von zB:
  M20="subModel1" P(0 0 0) DZ DIY



Erzeugen von internen Submodels:

- Menu subModels / move main - new submodel
    alle Objekte des aktiven Mainmodel in ein Submodel umkopieren.


- Menu subModels / create new submodel
    Ein neues Submodel anlegen und aktivieren.


- Menu subModels / load sub-model from file
    - ein gespeichertes (externes) Model in ein internes subModel laden
    - das interne model kann unabhängig vom gespeicherten Model modifiziert werden













M external Model


Benutzen von externen Submodels im CAD-Modus:

Ein externes Submodel positionieren.
  (Externe Submodels sind Modelfiles).
  
Submodel
  - Auswahl eines Modeldatei aus Liste (oder durch mittlere Maustaste )
  - oder durch Selektion eines bereits vorhandenen Submodels
  Modelname: siehe Symbolische Dateibezeichnungen.

Position,Orientation: Nullpunkt und Ausrichtung des Submodells

[Scale]: Vergrösserungsmaßstab; Default ist 1



Die Dateinamen der Modelfiles können symbolische Pfade und System-Variablen enthalten.
Externe Submodels werden intern zwischengespeichert,
  im Modellfile aber nicht abgespeichert.



Beispiel externes Model:
M20="Data/Niet1.gcad" R(P(100 0) DIY) VAL(0.5)
M21="Data/wsh14.stp" R(P(200 0))
M22="Data/obj43.wrl" R(P(0 0 0) R20)     # Mockup-Model



Benutzen von externen Submodels im MAN-Modus:
  durch Key-In von zB:
  M20="Data/Mod1.gcad" P(0 0 0) DZ DIY



Erzeugen von externen Submodels:

- Menu subModels / load submodel from file
    Ein Model aus einer Datei in eine neues (externes) Submodel laden.

- Menu subModels / save active submodel to file
    - ein internes-submodel wird in Datei gespeichert (wird zu externem model)










M CatalogPart

  
Benutzen von Normteilen im CAD-Modus:

  Einen Katalogteil positionieren.
  
  
CatalogPart
  - Auswahl von Katalog und Normteil aus Liste (oder durch mittlere Maustaste)
  - oder durch Selektion eines bereits vorhandenen Normteils ()


Position,Orientation
  Nullpunkt und Ausrichtung des Submodells

[Scale]: Vergrösserungsmaßstab; Default ist 1


  
Beispiel Katalogteil:
M22="Schrauben/SKS_10x30.ctlg" P20 VAL(1)



Benutzen von Normteilen im MAN-Modus:
  durch Key-In von zB:
  M28="Profile/6x4_150.ctlg" P (0 0 0) DZ DIY



Erzeugen von Normteilen:
  Siehe Normteile.
